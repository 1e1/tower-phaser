Chapter 1

Make a cannon move

We lay the foundations: how an image “comes to life”, how a cannonball follows a curve, and how we wrap it all up to share it.

1The game loop: 60 frames per second

A game is like a cartoon: a sequence of still images shown very fast. On screen, it gives the illusion of movement.

About 60 times per second, the browser calls a function we name update. On each call, we move the objects a little, then redraw. That's the game loop.

Why dt? Not all computers run at the same speed. By multiplying the movements by the elapsed time, the game runs at the same speed everywhere. Clever!

2“Scenes” like in a theatre

We split the game into scenes, a bit like the rooms of a house: the title screen, the settings screen, the match itself. We move from one to another.

  • 🏁 Boot — gets the game ready, the “press to play” screen.
  • ⚙️ Setup — we pick the names and the number of rounds.
  • 💥 Game — the duel.
  • 🏆 Result — who won?

Phaser handles this on its own: we give it the list, and we say “go to the Game scene”.

3The path of a cannonball

When the cannon fires, the ball goes in a straight line… then gravity pulls it down, and the wind pushes it sideways. The result is a nice curve: a parabola.

4Docker: the box to go

My game works on my computer… but will it work on yours? Often, a program needs specific tools (this version of that software). If they're missing, it crashes.

Docker fixes that: we put the game and all its tools into a “box” (a container). This box works the same everywhere: on your PC, on a server, in the cloud. A bit like a shipping container: whatever the ship, the box is always the same.

Remember. Batch 1 = a game that runs, inside a Docker box ready to be shared. Not pretty yet: that's the job of chapter 2!

?What is a “web game”?

A web game is a game that runs in a browser (Chrome, Firefox, Safari…) without installing anything. You click a link, and it plays!

For that to work, we put together three ingredients you may already know:

  • 🧱 HTML — the structure of the page (the “bricks”).
  • 🎨 CSS — the styling: colours, sizes, layout.
  • ⚙️ JavaScript — the brain: it's what makes things move.

Our game draws everything inside a special area called a canvas (a digital painter's “canvas”). We use a toolbox, Phaser, which handles the tricky parts (displaying, animating, listening to the keyboard) so we can focus on the game.

Remember. The browser already knows how to display pages. A web game is “just” a very lively page where JavaScript redraws the image dozens of times per second.

★The game we're building

Two towers, one on each side of a landscape. Each player sets the angle and the power of their cannon, taking the wind into account. The twist: both players aim at the same time, then both shots fire together. The first to hit the opposite tower wins the round!

We built the game in 4 steps (we call them “batches”). Each chapter tells one step, with its own biome mood:

BATCH 1 · Meadow

Make a cannon move

The game loop, the path of a cannonball, and how we “wrap up” the game with Docker.
BATCH 2 · Desert

Make it look good

Scenery (biomes), particles, depth (parallax) and sound effects made by code.
BATCH 3 · Tundra

Play across several screens

The TV displays, the phones control. A server referees the match in real time.
BATCH 4 · Volcano

Destroy the landscape

Like in Worms: every shot digs a crater. We'll see the magic trick to “punch a hole” in an image.

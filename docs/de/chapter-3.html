Kapitel 3

Über mehrere Bildschirme spielen

Der Fernseher zeigt das Match; die Handys dienen als Controller. Damit alle gleichzeitig dasselbe sehen, brauchen wir einen Schiedsrichter: den Server.

1Das Problem

Bisher passierte alles auf einem einzigen Computer. Jetzt: ein Fernseher, zwei Handys… drei verschiedene Geräte, die dasselbe Match erleben müssen, perfekt synchron.

Würde jedes Gerät das Match in seiner eigenen Ecke berechnen, kämen sie irgendwann durcheinander (eine Kugel hier, nicht dort…). Wir brauchen eine einzige Quelle der Wahrheit.

2Der Server ist der Schiedsrichter

Wir fügen ein Programm hinzu, das „in der Mitte“ läuft: den Server. Die Geräte sind Clients. Der Server ist der Einzige, der das Match berechnet (die Physik, die Punkte); die Clients zeigen nur an, was er entscheidet, und senden ihm die Befehle der Spieler.

  • 📱 Die Handys senden: „mein Winkel = 60°, ich bestätige“.
  • 🧑‍⚖️ Der Server berechnet die Salve, wer getroffen wird, den Punktestand.
  • 📺 Alle empfangen den Spielzustand und zeigen ihn auf dieselbe Weise an.
Merke dir. „Der Schiedsrichter hat immer recht“: Da der Server alles entscheidet, können sich die Bildschirme nicht widersprechen – und Schummeln wird schwieriger.

3Eine Leitung, die offen bleibt: der WebSocket

Eine klassische Webseite fragt nach einer Information, bekommt eine Antwort und legt dann auf. Zu langsam für ein Spiel! Wir benutzen einen WebSocket: eine Leitung, die in beide Richtungen offen bleibt. Der Server kann „sprechen“, wann immer er will, ~30-mal pro Sekunde.

Probier's aus! Klicke, um eine Salve abzufeuern, und sieh den Nachrichten beim Reisen zu: Die Befehle gehen hinauf zum Schiedsrichter, dann kommt der Zustand zu jedem Bildschirm zurück.

Der Tanz der Nachrichten

Blau/rot: die Befehle der Spieler. Grün: der Spielzustand, der an alle zurückgeschickt wird.

4Dieselbe Landschaft überall, ohne alles zu senden

Das Gelände kann aus tausenden Punkten bestehen: zu groß zum Senden. Der Trick: Der Server sendet nur einen kleinen zufälligen seed (eine Zahl). Jeder Bildschirm berechnet dieselbe Landschaft aus diesem seed neu (erinnerst du dich an den Generator aus Kapitel 2!).

5Einem Match beitreten: Raumcode & QR

Der Fernseher erstellt einen Raum und zeigt einen 4-Buchstaben-Code und einen QR-Code. Die Handys scannen den QR: Sie landen direkt im richtigen Match. Praktisch!

Für Neugierige. Der QR muss auf die lokale Netzwerkadresse des Fernsehers zeigen (so etwas wie 192.168.1.20), nicht auf „localhost“ (was „ich selbst“ bedeutet). Sonst würde das Handy sich selbst suchen! Der Server findet die richtige Adresse von allein.

6Wer macht was? Der Fernseher regelt das allein

  • 📺 Ein erkannter Fernseher startet sofort im „Host“-Modus: keine Fernbedienung nötig.
  • 🟢 Das Match startet von allein, sobald der 2. Spieler beigetreten ist.
  • 🧑‍🤝‍🧑 Weitere Spieler? Sie schauen zu, während sie auf einen frei werdenden Platz warten, und übernehmen dann.
Merke dir. Etappe 3 = der Wechsel von einem Spiel „auf einem Bildschirm“ zu einem „vernetzten“ Spiel, ohne alles neu zu schreiben: Wir haben nur die Berechnung zum Schiedsrichter verschoben.

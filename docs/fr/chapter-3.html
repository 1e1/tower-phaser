Chapitre 3

Jouer sur plusieurs écrans

La TV montre le match ; les téléphones servent de manettes. Pour que tout le monde voie la même chose en même temps, il faut un arbitre : le serveur.

1Le problème

Jusqu'ici, tout se passait sur un seul ordinateur. Maintenant : une TV, deux téléphones… trois appareils différents qui doivent vivre le même match, parfaitement synchronisés.

Si chaque appareil calculait le match dans son coin, ils finiraient par ne plus être d'accord (un boulet ici, pas là…). Il nous faut une seule source de vérité.

2Le serveur, c'est l'arbitre

On ajoute un programme qui tourne « au milieu » : le serveur. Les appareils sont les clients. Le serveur est le seul à calculer le match (la physique, les scores) ; les clients ne font qu'afficher ce qu'il décide et lui envoyer les ordres des joueurs.

  • 📱 Les téléphones envoient : « mon angle = 60°, je confirme ».
  • 🧑‍⚖️ Le serveur calcule la salve, qui est touché, le score.
  • 📺 Tout le monde reçoit l'état du jeu et l'affiche de la même façon.
À retenir. « L'arbitre a toujours raison » : comme le serveur décide de tout, les écrans ne peuvent pas se contredire — et tricher est plus difficile.

3Une ligne téléphonique qui reste ouverte : le WebSocket

Une page web classique demande une info, reçoit une réponse, puis raccroche. Trop lent pour un jeu ! On utilise un WebSocket : une ligne qui reste ouverte dans les deux sens. Le serveur peut « parler » quand il veut, ~30 fois par seconde.

À toi de jouer ! Clique pour lancer une salve et regarde les messages voyager : les ordres montent vers l'arbitre, puis l'état redescend vers chaque écran.

La danse des messages

Bleu/rouge : les ordres des joueurs. Vert : l'état du jeu renvoyé à tout le monde.

4Le même paysage partout, sans tout envoyer

Le terrain peut faire des milliers de points : trop gros à envoyer. L'astuce : le serveur n'envoie qu'une petite graine aléatoire (un seul nombre). Chaque écran recalcule le même paysage à partir de cette graine (souviens-toi du générateur du chapitre 2 !).

5Rejoindre un match : code de salon & QR

La TV crée un salon et affiche un code à 4 lettres et un QR code. Les téléphones scannent le QR : ils atterrissent direct dans le bon match. Pratique !

Pour les curieux. Le QR doit pointer vers l'adresse réseau locale de la TV (quelque chose comme 192.168.1.20), pas vers « localhost » (qui veut dire « moi-même »). Sinon le téléphone se chercherait lui-même ! Le serveur trouve la bonne adresse tout seul.

6Qui fait quoi ? La TV se débrouille seule

  • 📺 Une TV détectée démarre tout de suite en mode « hôte » : pas besoin de télécommande.
  • 🟢 Le match démarre tout seul dès que le 2e joueur a rejoint.
  • 🧑‍🤝‍🧑 Des joueurs en plus ? Ils regardent en attendant qu'une place se libère, puis prennent le relais.
À retenir. Lot 3 = passer d'un jeu « sur un écran » à un jeu « en réseau », sans tout réécrire : on a juste déplacé le calcul vers l'arbitre.
